# metacubexd Monorepo + Desktop + Server — Plan Index

> **For agentic workers:** This is a multi-plan effort. Execute the six plans **in dependency order** below, one plan at a time, each via superpowers:subagent-driven-development (recommended) or superpowers:executing-plans. Each plan is independently shippable and ends green.

**Spec:** [`docs/superpowers/specs/2026-06-14-monorepo-desktop-server-design.md`](../specs/2026-06-14-monorepo-desktop-server-design.md) — the single source of truth for types, the control-API route table, and shared contracts. Read it before any plan.

**Goal:** Turn the metacubexd Nuxt dashboard into a pnpm monorepo that ships the same UI as a hosted panel, an Electron desktop app, and an all-in-one Docker server — the latter two bundling and supervising their own mihomo kernel, with profile management and a Monaco/monaco-yaml config editor.

## Shared contracts (must hold across all plans)

These were reconciled during cross-plan review; each plan already encodes them. Summary for the executor:

- **Packages:** root `metacubexd-monorepo` (private); `@metacubexd/ui`, `@metacubexd/agent`, `@metacubexd/server`, `@metacubexd/desktop`.
- **`@metacubexd/agent` is source-exported** (no dist/CJS). Exports: `createAgent`, `createSupervisor`, `createProfileStore`, `createControlRouter`, types from `./types`, `MIHOMO_VERSION`/`mihomoAsset` from `./kernel/assets`, `fetchKernel` from `./kernel/fetch-kernel`.
- **Control API prefix is `/api/control`** everywhere (desktop in-process HTTP, server routes, Docker healthcheck `/api/control/health`). Auth: `Authorization: Bearer <token>` (SSE also accepts `?token=`); in-process desktop binding skips auth.
- **Kernel binding:** the agent's supervisor injects `external-controller`/`secret`/`mixed-port` into the active YAML before every spawn (C2). Consumers pass the pre-picked endpoint/secret into `createAgent` (server: `0.0.0.0:<CLASH_API_PORT>` + `CLASH_SECRET`; desktop: `127.0.0.1:<clashPort>` + random secret).
- **SSE frames are flat:** `{ type:'log', ... }` and `({ type:'state' } & KernelState)`.
- **Desktop bridge:** `window.metacubexd = { isDesktop, control:{base,token}, endpoint:{url,secret} }`; `packages/ui/plugins/desktop-endpoint.client.ts` (owned by plan 05) seeds the endpoint store.
- **Capability gating:** all new UI is gated on `GET /api/control/info`; on 404/error the dashboard behaves exactly as today (pure remote panel).
- **mihomo pin `v1.19.27`**, amd64 uses the `-compatible` asset, `.gz` is a raw binary (gunzip, never tar).
- **Ports/env (server):** `CONTROL_PORT=8080`, `CLASH_API_PORT=9090`, `MIXED_PORT=7890`, `CONTROL_TOKEN`, `CLASH_SECRET`, `DATA_DIR=/data`, `UI_DIST`, `MIHOMO_BIN`.
- **Tests:** vitest everywhere. `packages/ui` fakes Nuxt auto-imports via `test/setup.ts` — register any new Vue API / store dep used in a test there. Electron packaging / Dockerfile / CI yaml use "make change → verification command → commit" steps instead of red-green TDD.
